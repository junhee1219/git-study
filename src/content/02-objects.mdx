## "git이 그냥 마법 같아요"

신입이 가장 자주 하는 말이다. 명령어를 외워서 굴리긴 하는데 안에서 뭐가 벌어지는지 모르니 reset, rebase 같은 명령은 무서워서 못 쓴다.

그런데 막상 까보면 git의 데이터 모델은 **놀랍게 단순**하다. 챕터 1에서 본 "스냅샷"은 사실 단 3종류의 객체로 표현된다 — **blob**, **tree**, **commit**. 이 챕터 끝나면 `.git/` 디렉토리 안이 손바닥처럼 보일 거다.

<Callout tone="success" title="이 챕터의 한 줄 요약">
  git은 3종류의 객체(blob · tree · commit)를 SHA-1 해시로 식별해서 저장한다. 같은 내용 = 같은 해시 = 한 번만 저장.
</Callout>

## 3종 객체

| 객체 | 역할 | 비유 |
|---|---|---|
| **blob** | 파일의 **내용** (이름은 없음) | 사진 1장 |
| **tree** | 디렉토리 — `(이름, 해시, 모드)` 목록 | 사진들이 담긴 폴더 + 목차 |
| **commit** | 루트 tree 한 개 + 부모 commit 해시 + 메시지/작성자/시간 | 그 순간 폴더 전체에 붙인 라벨 |

세 가지가 다 SHA-1 해시(40자)로 식별된다. 해시는 객체 내용으로부터 계산된다 — 그래서 **같은 내용이면 같은 해시**, **다른 내용이면 다른 해시**가 나온다 (충돌 가능성은 천문학적으로 낮음).

### 각 객체가 가리키는 방향

```
commit ──→ tree ──→ blob (파일 내용)
   │         │
   │         └─→ tree (하위 디렉토리) ──→ blob, tree, ...
   │
   └─→ parent commit (이전 시점)
```

핵심은 **단방향 참조**다. commit은 tree를 가리키고, tree는 blob을 가리킨다. 반대 방향은 없다. blob은 자기가 어떤 파일 이름인지조차 모른다 — 이름은 tree가 들고 있다.

## 직접 만져보기

세 파일을 편집해보자. 내용이 바뀌면 그 자리에서 **blob 해시가 바뀌고**, tree 해시도 따라 바뀌고, commit을 누르면 새 commit 객체가 만들어진다.

특히 `README.md`와 `COPY.md`의 내용이 **완전히 같다는 점**에 주목. 같은 blob 하나를 두 파일이 공유한다 — 이게 챕터 1의 "왜 용량이 폭발 안 하느냐" 질문의 답이다.

<ObjectInspector />

<Callout tone="info" title="관찰 포인트">
  1. <code>README.md</code>를 한 글자만 바꿔보자 → 두 파일의 blob 해시가 갈라진다.
  2. 두 파일을 다시 똑같이 만들어보자 → 다시 같은 blob을 공유한다.
  3. commit 두 번 찍으면 두 번째 commit이 <code>parent</code>로 이전 commit 해시를 가리킨다 — 이게 히스토리의 사슬.
</Callout>

명령어 세 개 — `git cat-file -p <hash>`, `git ls-tree HEAD`, `git rev-parse HEAD` — 만 알아도 `.git/` 안을 자유롭게 탐색할 수 있다.

## 함정과 팁

- **함정 1: "commit 메시지만 바꾸면 hash가 그대로일 줄 알았어요"**
  commit hash는 tree + parent + 메시지 + 작성자 + 타임스탬프 **모두**의 해시다. 메시지 한 글자만 바꿔도 hash가 통째로 바뀐다. `git commit --amend`가 새 commit을 만드는 이유다.

- **함정 2: "같은 파일 두 개면 용량 두 배 아닌가요?"**
  아니다. blob은 **내용** 기반이라 자동으로 deduplication된다. 위 예제에서 `a.txt`와 `b.txt`가 같은 blob을 공유한 게 그 증거.

- **함정 3: "blob 안에 파일명도 들어있나요?"**
  들어있지 않다. blob은 순수 내용. 파일명은 tree가 들고 있다. 그래서 같은 내용을 다른 이름으로 두 군데 둬도 blob은 하나.

- **팁: `git log --oneline`의 짧은 해시도 객체 해시의 앞 글자**
  git이 알아서 충돌 안 나게 잘라서 보여줄 뿐, 실제론 40자 SHA-1이다.
